HiTIDE, the High-level Tool for Interactive Data Extraction, allows users to subset and download popular PO.DAAC level 2 (swath) datasets. Users can search across a wide variety of parameters, such as variables, sensors and platforms, and filter the resulting data based on spatial and temporal boundaries of interest to the user. HiTIDE goes even further, offering instant previews of variable imagery, allowing users to rapidly find data of interest for download and further, rigorous scientific analysis.

Why are there cloud icons () next to certain datasets?
HiTIDE is migrating to the cloud and is also shifting to cloud based services for retrieving metadata and performing subsets on datasets.

This migration is taking a phased approach where datasets will be transitioned in groups. Therefore, during the transition, you will be able to see both pre-transition (classic) datasets and post-transition (cloud) datasets in the dataset list. Cloud datasets can be identified by the presence of the icon.
As a HiTIDE user, does it make any difference if a dataset has transitioned to the cloud services or not?
Here are the current differences between Classic Datasets (which have not yet transitioned to cloud services) and Cloud Datasets:

Classic Datasets Cloud Datasets
Download links work for 48 hours. Download links work for 30 days.
An email with download links is sent when the subset job is complete. The download links are also available in HiTIDE, in the "Your Downloads" section, under the "Downloads" tab. Currently, no email is sent when the subset job is complete. Download links can only be obtained from the HiTIDE client.
When multiple granules are subsetted, the download link is to a zip file that includes all the resulting granules. Each resulting granule gets its own download link.
Cut Scanline feature is optional and can be toggled on or off when submitting a subset job (see the Cut Scanline part of the FAQ for more information). Cut Scanline feature is always applied when subsetting cloud datasets. The Cut Scanline toggle currently only affects classic datasets.
There is no granule limit. Current limit is 200 granules per subset. If you submit a job with more than 200 granules, the subset will still succeed, but the result will only contain 200 granules.
What is Earthdata Login?
NASA's Earth Observation System Data and Information System (EOSDIS) Earthdata Login provides a simple, centralized mechanism for user registration and user profile management for all Distributed Active Archive Center (DAAC) users accessing data.
Why do users have to register?
The information provided by users when they register for Earthdata Login helps NASA better understand how our data are being used and the diverse needs of multiple user communities. This helps us prioritize improvements in data discovery, access, and usability. Registration also allows us to inform users with pertinent information such as the availability of new or updated data products, or data quality issues. In the future, it will allow for new value-added features and customized services resulting in a better user experience. For questions regarding Earthdata Login, see https://urs.earthdata.nasa.gov/documentation.
What does the "Cut Scanline" download option mean?
Swath data are often captured in scanlines. Because HiTIDE allows users to specify custom regions of interest, this presents a dilemma when subsetting the data. Some users want only the data within their region of interest, and nothing else. Other users want the entire scanline, regardless of if it all falls within their region of interest. To this end, HiTIDE allows the option to “cut scanlines” at the granule download screen.

Not cutting scanline conserves the original scanline width.

The red box is the bounding box used to subset, and the blue boxes are the scanlines that intersect with the red box. The data in the blue boxes within the red box will be including in the subsetted result. The data in the blue boxes outside of the red box will be changed to No Value and included in the subsetted result to conserve the original scanline width.

Cutting scanline could reduce the original scanline width to make the file size smaller.

The red box is the bounding box used to subset, and the blue boxes are shortened scanlines that intersect with the red box. The data in the blue boxes within the red box will be including in the subsetted result. The data in the blue boxes outside of the red box will be changed to No Value and included in the subsetted result. The original scanline width has been shortened to reduce the data size.
What does the "Merge Granules" download option mean?
By enabling the "Merge Granules" download option in the "Downloads" tab, all of the subsetted granules for a given job will be merged into a single NetCDF4 granule. This granule will have an additional index called "subset_index" added to the variables. This index maps the file from which the subsetter portion of the data comes. For example, if you have 50 files in your subset region, merging them will give you a single file back with an "subset_index" (50) for each variable found in the original files. This file is meant to be easier to work with instead of multiple individual files.

  • All global metadata common to the files such as dataset IDs will be added to the merge while metadata that changes from file to file, like coverage start and stop times, will be left out.
  • The PO.DAAC dataset Persistent ID will be added to the metadata as "PODAAC-persistent-ID"
  • The "request-start-time", "request-end-time", and "request-bounding-box" parameters will be added to the merge file. These items should match the query you performed to find the data.
  • The "time_coverage_start" and "time_coverage_end" parameters will be added to the global metadata. These values represent the earliest and latest time for which data was found in the original files. Your subsetted data will fall between these bounds.
  • Merged variables are filled with a "fill_value" if there are no data for a given index into the variable array or if the data from which the subset came are the fill value.

Where are configurations saved?
Configurations are saved in LocalStorage in your browser and will persist indefinitely unless manually cleared through browser settings. Note that although LocalStorage space is limited to around 5MB per website, HiTIDE configurations are relatively small, meaning up to ~5000 configurations can be saved.
Why are some variable previews not appearing?
Most of the time when a user clicks on the variable preview icon, the variable image appears immediately. There are times, however, when no image appears. There are several reasons this may happen, but most often this is because there are no valid data within the file being imaged. This can happen when the sensor is over land, or cloud cover interferes with the measurement, or many other possibilities. If you think an image not being available is in error, please contact us through the PO.DAAC forum and we will explore the issue.
How does HiTIDE handle subsetting and extraction?
HiTIDE downloads work by submitting the searches, filters, and other parameters to a download manager service. This service grabs all of the relevant data files to a staging area, and then removing all of the data falling outside of the optionally defined bounding box. Depending on the options the user selects, all or some of the data variables can be included in the download. For each file in the download, a corresponding “subsetted-*“ file is generated in the native file format of the original granule. The resulting files are packaged and compressed, delivered as a ".zip" file to the user.
Why are some completed download links returning 404s?
Users experiencing this issue are probably attempting to download data that is more than 48 hours old. Downloads are available for up to 48 hours, and after that are purged to make space for new job submissions. Download jobs are listed in the HiTIDE interface until the user removes them by clicking the ‘X’ next to the download link.
Variable preview disclaimer:
It is very important for the users of HiTIDE to be aware that the variable image previews are just that: previews. No quality flags have been applied to the data. Variable imagery is designed to be a quick look product, allowing users to view a region of interest at a coarse level. Many times, quality flags are imaged as well so some degree of accurate coverage can be deduced, but HiTIDE is designed to be a “search and subset” service, where users ultimately download and analyze their own data.
